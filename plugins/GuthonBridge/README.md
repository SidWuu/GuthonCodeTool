# Guthon Bridge

Guthon Bridge 是面向 Guthon 在线开发页面的本地辅助扩展，由 Chrome 扩展和本地 HTTP 桥接服务组成。

## 功能

- 在过程函数开发页识别当前脚本，并保存为本地文件。
- 在模块开发页打开复制模式，按页面块展示字段信息，便于复制和整理。
- 在模块开发页的字段平移按钮组中复制字段并粘贴到另一个已选组件。
- 在数据表管理页拉取当前数据源的表结构，支持只拉取选中的数据表。
- 在单据类型页拉取当前数据源的单据类型，支持只拉取选中的单据类型。
- 通过本地 HTTP 服务保存文件，并记录远端对象、本地路径和版本元数据。
- 可选调用 Guthon Hub，从源码表拉取已存入源码并生成工作副本。

## 目录

```text
plugins/GuthonBridge/
  bridge/       本地 HTTP 桥接服务
  extension/    Chrome 扩展
  package.json  启动和测试脚本
```

## 启动 Bridge

```bash
npm run start:bridge
```

默认监听：

```text
http://127.0.0.1:17361
```

健康检查：

```text
http://127.0.0.1:17361/health
```

临时切换端口：

```bash
GUTHON_BRIDGE_PORT=17362 npm run start:bridge
```

Chrome 扩展默认访问 `127.0.0.1:17361`，改端口后需要同步调整扩展代码或权限配置。

## 安装 Chrome 扩展

1. 打开 `chrome://extensions`
2. 开启“开发者模式”
3. 点击“加载已解压的扩展程序”
4. 选择 `plugins/GuthonBridge/extension/`

修改扩展代码后，需要在 Chrome 扩展页重新加载扩展，并刷新开发平台页面。

扩展弹窗右上角提供关闭按钮；页面内悬浮按钮可拖动，位置会保存在当前浏览器页面环境中。

## 允许的开发平台地址

编辑 `extension/host-config.js` 顶部配置后重新加载扩展：

```js
const config = {
  protocols: ["http:", "https:"],
  ipRanges: ["192.168.0.0/16", "192.0.2.0/24"],
  domainSuffixes: ["dev.example.com"],
  pathPrefixes: ["/guthon/"]
};
```

`ipRanges` 使用 IPv4 CIDR。`domainSuffixes` 同时匹配配置域名及其子域名，例如 `dev.example.com` 也允许 `team.dev.example.com`。

## 使用

### 拉取页面当前源码

1. 启动本地 bridge。
2. 打开过程函数开发页面。
3. 点击 Chrome 工具栏中的 `Guthon Bridge` 图标。
4. 确认弹窗识别到包名和函数名。
5. 填写本机绝对页面保存目录。
6. 点击 `拉取页面当前源码`。

过程函数页面上的 `源码拉取` 按钮不使用页面保存目录，会直接走源码表版本拉取。

### 拉取源码表版本

1. 启动本地 bridge。
2. 打开过程函数开发页或模块开发页。
3. 点击 `拉取源码表版本`。

拉取成功后会显示工作副本路径。若本地修改已回写到远端，会刷新工作副本基线；若本地与远端内容分叉，按钮会显示冲突提示并保留本地内容。

### 拉取表结构

1. 启动本地 bridge。
2. 打开数据表管理页。
3. 选择数据源。
4. 可选：选中一个或多个数据表。
5. 点击页面悬浮按钮或 Chrome 扩展弹窗中的 `拉取表结构`。

弹窗会显示当前数据源的编码和名称。未选中数据表时，拉取当前数据源下全部表结构；选中数据表时，只拉取选中的表结构。

### 拉取单据类型

1. 启动本地 bridge。
2. 打开单据类型页。
3. 选择数据源。
4. 可选：选中一个或多个单据类型。
5. 点击页面悬浮按钮或 Chrome 扩展弹窗中的 `拉取单据类型`。

弹窗会显示当前数据源的编码和名称，以及当前选中的单据类型编码。未选中单据类型时，拉取当前数据源下全部单据类型；选中单据类型时，只拉取选中的单据类型。

### 打开复制模式

1. 打开模块开发页面。
2. 点击页面上的 `复制模式` 按钮，或点击 Chrome 扩展弹窗中的 `打开复制模式`。
3. 在弹窗中查看字段结构。
4. 从右侧文本框复制纯文本内容。

### 字段平移

1. 在模块开发页面选中来源组件，点击 `复制字段`。
2. 在弹窗中勾选字段并点击 `复制`。
3. 选中目标组件，点击 `粘贴字段`。

仅支持输入组件的字段和表格组件的列。粘贴按 `fieldId` 跳过重复项；缓存只保留到页面刷新前，平台保存仍需由用户单独执行。

## 本地保存规则

- 保存目录必须是本机绝对路径。
- 保存目录不存在时会自动创建。
- 文件名来自当前函数名或片段名。
- 文件名会过滤路径非法字符。
- 同名文件会覆盖。
- 文件直接保存到填写的目录下，不再按包名创建子目录。

本地映射文件：

```text
bridge/workspace/manifest.json
```

## 不支持

- 本地文件回推到开发平台。
- 自动签出过程函数。
- 自动签出页面源码。

收到签出或回推命令时，插件会直接返回错误，不调用平台保存接口。

## 已知限制

- 复制模式依赖开发平台页面 DOM 和 Vue 实例结构，平台页面结构大改时需要同步调整。
- 模块开发主页面中复杂 tab/form/table 组合仍在继续兼容。
- 保存目录只能手动输入，还没有目录选择器。
